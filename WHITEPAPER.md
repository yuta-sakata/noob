## 摘要

威妥玛式拼音编程是一种创新的编程范式，通过使用威妥玛拼音系统（Wade-Giles romanization）作为编程语言关键字，为中文母语程序员提供更自然、更符合认知习惯的编程体验。菜鸡语言作为首个完整实现威妥玛式拼音编程理念的解释型编程语言，继承自灵雀语言的核心架构，证明了这一范式在实际编程中的可行性与优势。

## 核心理念

威妥玛拼音（Wade-Giles romanization）由英国外交官威妥玛于1859年创制，是19世纪末至20世纪中期最主要的汉语罗马化系统。其拼写规则采用连字符分隔音节（如 `pien-liang`、`han-shu`），元音系统清晰直观，声调标记可选。在编程应用中，威妥玛式拼音展现出独特优势：连字符天然分隔音节避免歧义，音节边界清晰提升可读性，ASCII字符集完全支持保证国际化兼容，且相比汉语拼音的声调标记更为简洁优雅。

菜鸡语言将威妥玛拼音作为编程关键字，实现了认知自然性、文化适配性与技术严谨性的完美融合。使用 `pien-liang`（变量）代替 `var`，`han-shu`（函数）代替 `function`，使代码阅读如同阅读拼音文本，大幅降低中文母语者的学习曲线。同时，威妥玛拼音承载着中华文化的历史记忆，在编程中使用它是对文化传承与技术创新的结合。在技术层面，菜鸡语言采用静态类型系统、完整的词法和语法分析器，以及模块化的解释器架构，威妥玛拼音只是表层语法，底层保持现代编程语言的所有严谨性。

## 技术实现

菜鸡语言采用纯拼音关键字设计，所有语言关键字使用威妥玛式拼音（如 `ju`、`tang`、`wei`、`fan-hui`），类型系统包含 `cheng`(整型)、`fu`(浮点)、`chuang`(字符串)、`pu`(布尔)等拼音类型，内置函数库提供 `ta-yin-hang`(打印行)、`chang-tu`(长度)、`chieh-pien`(切片)等拼音命名函数。

实现架构遵循现代编程语言的标准流程：词法分析→语法解析→AST构建→解释执行，分别对应威妥玛拼音词法、递归下降解析、类型检查和模块化执行。设计上支持连字符分隔的拼音标识符，不支持撇号以避免解析复杂性，完整支持数组、枚举、递归、静态存储等现代编程特性。

## 应用价值

在教育领域，威妥玛式拼音编程可降低中文母语初学者的入门门槛，实现双语编程教学，在技术学习中融入文化传承。在工程实践中，适用于快速原型开发、为中文用户定制DSL、实现文档化编程（代码本身即为拼音文档）。在学术研究方面，为探索非英语编程语言设计、研究母语编程对认知负荷的影响、推动编程语言多样性和文化适配性研究提供了重要案例。

## 实践案例

```noob
// 斐波那契数列示例
han-shu fei-po-na-chi(pien-liang n:cheng):cheng {
    ju (n <= 1) {
        fan-hui n;
    }
    fan-hui fei-po-na-chi(n - 1) + fei-po-na-chi(n - 2);
}

han-shu main():kung {
    wei (pien-liang i:cheng = 0; i < 10; i++) {
        ta-yin(fei-po-na-chi(i), " ");
    }
    ta-yin-hang();
}
```
